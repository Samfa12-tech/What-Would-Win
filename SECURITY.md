# Security policy

## Reporting a vulnerability

Use GitHub's private vulnerability-reporting feature for this repository when it is available. Do not include credentials, private user data, unpublished exploit details or other sensitive material in a public issue.

If private reporting is unavailable, open a minimal public issue asking the maintainer to establish a private channel. Do not include the sensitive details in that issue.

## Supported version

What Would Win is currently a pre-release static web application. Security fixes apply to the latest revision of the default branch.

## Scope

Useful reports include unsafe handling of imported or shared data, browser-storage privacy failures, script or URL injection, dependency compromise indicators, and accidental publication of credentials or private information.

The deterministic simulation's calibration choices and entertainment-model outcomes are product/model questions rather than security vulnerabilities.
